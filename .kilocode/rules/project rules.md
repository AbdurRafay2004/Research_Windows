## Guidelines

Project Rules
Rule 1: Continuous Documentation & Progress Tracking
- CHANGELOG.md is the single source of truth.
- After completing any significant task, feature, or architectural change, immediately update it with:
- Current Status – Summary of the latest milestone (e.g., "✅ Phase 1 Complete").
- ChangeLog – Add every change in timeline order (newest at the bottom).
- Immediate Next Steps – Updated, prioritized to-do list.
- Known Issues/Notes – Technical debt, bugs, or constraints discovered.
Rule 2: README.md Maintenance
- Keep README.md up to date with:
- Project introduction
- Tech stack
- Project structure
- Key decisions
- Setup/usage instructions
Rule 3: Environment
- Development is done on Windows 11.
Rule 4: Documentation Reference
- Use context7 MCP for up-to-date libraries and documentation when needed.
Rule 5: Best Practices
- Maintain BestPractices.md to store and reference best practices for the project.
- Create the file if it does not exist
